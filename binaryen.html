WebAssembly (wasm) is a new binary executable format

Will allow even large compiled codebases to run efficiently on the web

Main initial focus is on C and C++, since those are very popular (e.g., in the games space: Unity, Unreal, etc.)

Using emscripten, currently two options:

C++  ⇒  clang  ⇒  asm.js backend  ⇒  asm2wasm  ⇒  wasm

C++  ⇒  clang  ⇒  wasm backend  ⇒  s2wasm  ⇒  wasm

asm2wasm and s2wasm are part of the binaryen project

binaryen handles wasm-specific tasks like binary generation and "compression" operations that might not make sense to do in LLVM

wasm backend avoids the limitations of the asm.js backend

Full i64 support

Supports all LLVM IR (full legalization)

LLVM's codegen can emit very good code

Developed upstream

However, wasm backend is slower to compile, mainly due to known issues about LLVM codegen (e.g., WebKit's recent switch from LLVM to B3)

Power of LLVM codegen comes at a cost

Incremental compilation will help both backends, but proportionally, so difference will remain

Similar tradeoffs appear, and perhaps more strongly, for Other languages => WebAssembly
LLVM is big and powerful [wun wun, mountain]
But sometimes a more nimble approach is what you want [arya, bronn, oberyn, syrio forel]
We saw c++ => clang => LLVM codegen => binaryen => wasm What if we compiled straight to binaryen...?
Compiling straight to Binaryen benefits * much smaller than LLVM * designed for parallel and pipelined codegen and optimization (optimize in multiple threads while still generating functions) * simple, compact data structures (e.g. no parent pointers, funcs have no global references, no use lists, similar philosophy to the B3 JIT) * simpler IR than LLVM (e.g. no arbitrary-sized ints) * Binaryen is a good way to emit wasm, so using it as your JIT means you avoid multiple IRs * Much of LLVM's power is irrelevant anyhow to a target like Binaryen (no need for regalloc, no need for ISA quirk tricks, etc.) * ??LLVM's codegen has known speed issues (WebKit left LLVM, various JIT projects failed, etc.)??
compile times, code size, benchmarks
Also it is very simple, there is a C API [code example] provide input in structured form or as basic blocks (which will be relooped) provide input in SSA form
or register form
