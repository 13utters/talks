WebAssembly overview and importance
WebAssembly is primarily a compiler target
C++ to WebAssembly asm2wasm is ready LLVM wasm backend is coming
why wasm backend? i64, full legalization but downsides too - compile time in particular, we go through binaryen anyhow, since some stuff is more at the "compression" layer, not for LLVM to do/hard to do there
Other languages => WebAssembly
LLVM is big and powerful [wun wun, mountain]
But sometimes a more nimble approach is what you want [arya, bronn, syrio forel]
We saw c++ => clang => LLVM codegen => binaryen => wasm What if we compiled straight to binaryen...?
Compiling straight to Binaryen benefits * much smaller than LLVM * designed for parallel and pipelined codegen and optimization (optimize in multiple threads while still generating functions) * simple, compact data structures (e.g. no parent pointers, no use lists, similar philosophy to the B3 JIT) * Binaryen is a good way to emit wasm, so using it as your JIT means you avoid multiple IRs
