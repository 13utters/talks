Browsers to support WebAssembly natively


Compilers that emit WebAssembly

Binaryen


A WebAssembly infrastructure library, written in C++


Represent and process WebAssembly

Compile asm.js to WebAssembly

Interpret WebAssembly

pretty printing colors

wasm.js


wasm.js is a port of the various Binaryen components to JavaScript, plus JavaScript-WebAssembly integration code

For example, it can execute WebAssembly in an interpreter in JavaScript, as a (slow) polyfill

Emscripten+Binaryen passes the Emscripten test suite, when running the output WebAssembly in the Binaryen interpreter

Good confidence:

Binaryen interpreter passes the WebAssembly spec test suite

Emscripten test suite is very comprehensive

Relatively little new code here, on top of a stable toolchain

Summary


Emscripten+Binaryen provide a usable C++ to WebAssembly compiler toolchain today: WebAssembly will not be delayed due to toolchain issues

Binaryen aims to fill the niche of "the WebAssembly library", helping basically anything that needs to work with WebAssembly code, like experimentation, polyfilling, etc.
